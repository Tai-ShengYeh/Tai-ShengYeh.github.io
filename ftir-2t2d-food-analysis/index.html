Concept

2T2D-COS 在這裡扮演什麼角色？

從一維到二維

原始 FTIR 是一條曲線：每個 wavenumber 對應一個吸收強度。2T2D-COS 把兩條光譜，例如「class mean spectrum」與「單一樣本 spectrum」，轉成一張二維 correlation map。

讓模型看到交互關係

一般 PCA / PLS-DA 直接看各波數的強度；2D map 則呈現 band-to-band 的共同變化。這可作為 CNN 或 image-based ML 的輸入，也可用來討論哪些區域共同貢獻分類。

教學版 two-trace 計算流程

Reference trace選一條參考光譜，例如 Arabica 的平均光譜，或全部訓練資料的平均光譜。
Sample trace取單一樣本光譜，與 reference 組成 2 × p 的矩陣，p 是波數點數。
Synchronous map觀察兩條光譜在波數對之間的共同變動；適合入門視覺化。
Asynchronous map使用 Hilbert–Noda matrix 產生相位/順序差異的資訊；可作為進階討論。
科學提醒：正式投稿時要清楚說明採用的 2D-COS / 2T2D 公式、前處理與 reference 選法。本頁的目標是教學與可重現練習，因此會先用最容易理解的 two-trace implementation。
Python workflow

學生可以直接改成 notebook 的 Python 骨架

下面程式會下載 Mendeley zip、讀取 coffee FTIR CSV、整理成 X: samples × wavenumbers，再產生每個樣本相對於所屬類別平均光譜的 2T2D-COS maps。

import io, zipfile, requests
import numpy as np
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.model_selection import StratifiedKFold, cross_val_score
from sklearn.svm import SVC

url = "https://data.mendeley.com/public-api/zip/frrv2yd9rg/download/1"
r = requests.get(url, headers={"User-Agent": "Mozilla/5.0"})
r.raise_for_status()
z = zipfile.ZipFile(io.BytesIO(r.content))

coffee_file = [f for f in z.namelist() if "instant_coffee" in f and f.endswith(".csv")][0]
raw = pd.read_csv(z.open(coffee_file), header=None)

sample_id = raw.iloc[0, 1:].astype(str).to_numpy()
group_code = raw.iloc[1, 1:].astype(str).to_numpy()
y = raw.iloc[2, 1:].astype(str).to_numpy()      # Arabica / Robusta
wavenumber = pd.to_numeric(raw.iloc[3:, 0]).to_numpy()
X = raw.iloc[3:, 1:].T.astype(float).to_numpy() # samples × wavenumbers

print(X.shape, y[:5], wavenumber.min(), wavenumber.max())

產生 2T2D maps

def noda_matrix(m):
    N = np.zeros((m, m))
    for j in range(m):
        for k in range(m):
            if j != k:
                N[j, k] = 1 / (np.pi * (k - j))
    return N

def two_trace_2dcos(sample, reference):
    Y = np.vstack([reference, sample])
    Y = Y - Y.mean(axis=0, keepdims=True)
    N = noda_matrix(Y.shape[0])
    sync = Y.T @ Y / (Y.shape[0] - 1)
    async = Y.T @ N @ Y / (Y.shape[0] - 1)
    return sync, async

class_mean = {c: X[y == c].mean(axis=0) for c in np.unique(y)}
sync_maps, async_maps = [], []
for xi, yi in zip(X, y):
    s, a = two_trace_2dcos(xi, class_mean[yi])
    sync_maps.append(s)
    async_maps.append(a)

print(sync_maps[0].shape)
Orange workflow

如果用 Orange Data Mining，先走一維光譜基礎流程

Orange 不一定要直接做 2D-COS。教學上可以先讓學生理解光譜矩陣與分類，再用 Python 產生 2T2D maps。這樣比較不會把「資料整理」與「2D correlation」兩個難點混在一起。

File→Select Columns→Preprocess→PCA→Scatter Plot→Test & Score→Confusion Matrix

入門問題

Arabica 與 Robusta 在 PCA score plot 是否自然分開？哪幾個 wavenumber 對 PC loading 影響最大？

中階問題

比較 raw、SNV、standard scaling、Savitzky–Golay derivative 對分類表現的影響。

進階問題

把 Python 產生的 2T2D maps 輸出成影像或 flattened features，再比較 SVM / Random Forest / CNN。
