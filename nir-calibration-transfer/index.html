NIR Calibration Transfer 教學：Corn 三儀器資料集

Python JupyterGoogle ColabOrange Data MiningPLS / DS / PDS / SBC

教學定位

這份教材用公開的 Eigenvector Research corn NIR dataset 示範 calibration transfer：同一批 80 個玉米樣品在三台 NIR 儀器 m5、mp5、mp6 上量測，波長 1100–2498 nm，每 2 nm 一點，共 700 channels，並提供 moisture、oil、protein、starch 參考值。

學生會先建立 master instrument（m5）的 PLS protein 模型，再直接套用到 slave instrument（mp5），觀察誤差惡化，接著用 SBC、DS、PDS 比較轉移效果。

為什麼這個 dataset 適合 calibration transfer？

條件此資料集狀態教學意義

Python 核心流程

  1. 下載 corn.mat 並解析 m5/mp5/mp6 spectra。
  2. Savitzky–Golay smoothing + SNV 作為基礎前處理。
  3. 用 m5 前 50 個樣品建立 PLS protein 模型。
  4. 用 10 個 paired standards 學習 slave→master 的轉換。
  5. 用 mp5 後 20 個樣品作 independent slave test。
  6. 比較 no transfer、SBC、DS、PDS 的 RMSEP 與 R²。
python analysis_script.py

教學注意事項

  • 這是 calibration transfer 教學，不是追求最佳 corn protein 模型。
  • SBC 需要 standard samples 的 reference y；DS/PDS 通常只需要 paired spectra。
  • PDS 對 transfer standards 的數量、代表性、window size 很敏感；標準樣品太少可能過度配適。
  • 不要把同一 sample 的不同 instrument spectra 當成獨立樣品隨機切分，否則會混淆 transfer 問題。

Orange Data Mining 課堂活動

  1. 安裝 Orange 與（可選）Orange Spectroscopy add-on。
  2. File → 載入 corn_nir_for_orange.csv。
  3. Select Columns：wl_* 為 features，Protein 為 target，sample_id 與 instrument 為 meta。
  4. PCA：用 instrument 上色，觀察三台儀器造成的光譜分布差異。
  5. 用 Select Rows 分出 m5 與 mp5，比較模型跨儀器預測落差。
  6. 把 Python 產生的轉移後 spectra 匯回 Orange，可視覺化 PDS 前後變化。
